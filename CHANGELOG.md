# 更新记录 / Changelog

本项目版本号遵循 `主.次.修订`。日期为发布当天（本地时区）。

## [1.0.3] - 2026-06-24

> 悬浮球拖拽体验优化：拖出即展开。

### 修复

- **悬浮球拖出后不展开**：从边缘拖出悬浮球松手后，之前会保持球形态漂浮或吸附回边缘，而非展开为卡片。修复后拖出松手直接展开卡片；吸附判定逻辑保持不变（距边缘 72px 内仍触发贴边），卡片拖动收球逻辑也不变。

## [1.0.2] - 2026-06-23

> 代码审查收敛版 + 热修：`invalidateQueries` 键值不匹配导致时间轴不刷新、Widget 跨窗口通信。

### 修复

- **时间轴不刷新**：TanStack Query v5 的 `invalidateQueries({ queryKey: ["tasks"] })` 默认 `exact: true`，只匹配精确键 `["tasks"]`，无法命中 `["tasks", "all"]`、`["tasks", "completed"]` 等带二级键的查询。修复：全部 3 处失效点（`ChatPanel`、`useTasks`、`CompletedModal`）显式加 `exact: false`。
- **Widget 不即时刷新**：主窗口与悬浮窗各持独立 `QueryClient`，主窗口的 `invalidateQueries` 无法让悬浮窗重新拉取数据，最长延迟 20 秒。修复：新增 `BroadcastChannel` 跨窗口通信（`lib/channel.ts`），主窗口任务变更时即时通知 Widget 侧 `invalidateQueries`。

### 安全

- **CORS 收敛为 allowlist**：服务端不再反射任意 `Origin`，仅放行开发期来源（`TASKDECK_ORIGIN`，默认 `localhost:5173`）与 Tauri webview（`tauri://*`、`tauri.localhost`）。不放行来源不回 `Access-Control-Allow-Origin`、预检直接 `403`——挡掉浏览器中任意网页对本机 loopback 任务库的跨源写操作。

### 改进

- **任务可改截止时刻**：`PATCH /tasks/:id` 字段白名单补入 `due_time`，为后续任务编辑 UI 铺路（此前只能改 `due_date`，时刻会被静默丢弃）。

### 文档

- 三份文档对齐现状：`api.md`（端点全集、`/health` provider 为 `deepseek`、删除已下线的 `TASKDECK_PROVIDER/MODEL` 环境变量）、`data-model.md`（补 `due_time`/`completed_at`、`category` 标 deprecated、新增 `tag_defs`/`settings` 表）、`architecture.md`（AI 降级 tag 修正为「待整理」）。
- 代码注释清理：统一降级 tag 措辞为「待整理」、去除 `schema.ts` 残留的 Agent SDK 字样。

## [1.0.1] - 2026-06-23

### 修复

- **窗口关闭改为隐藏**：点击 macOS 窗口 X 按钮不再销毁窗口，改为隐藏。修复关闭后 Dock/Launchpad 点击图标无法恢复、悬浮窗放大按钮失效的问题。Rust 层 `on_window_event` 拦截 `CloseRequested` 并调用 `api.prevent_close()` + `window.hide()`，`RunEvent::Reopen` 处理 Dock 点击恢复。

### 改进

- **AI 优先级标准对齐**：AI 给任务打优先级时，现在使用与自动升级相同的 24h/48h/72h 时间阈值（≤24h→急、≤48h→高、≤72h→中），作为**底线**；语义重要度可在此基础上**上调**（如「非常重要」的任务即使 DDL 远也可打高优先级）。消除 AI 初次分配与 5 分钟后自动升级之间的不一致。

## [1.0.0] - 2026-06-23

> TaskDeck 首个正式版。核心能力：自然语言一句话建任务（支持一次多任务）、AI 推断标签 / 优先级 / 相对时间、优先级随 DDL 实时升级、迫近高亮与系统提醒、四视图 + 桌面悬浮球。

### 新增

- **一次安排多个任务**：一句话可拆出多个任务——含多件事、或「X 号到 Y 号每天 / 每周」这类日期范围与重复，AI 逐一拆成多条入库，对话里一并展示（「已登记 N 项 ✓」）。如「7月1号到5号每天中午12点健身」→ 自动建 5 条。
- **日历「完成日」图片**：某天命中的任务**全部完成**时，该日数字变成一张随机图片（首次确定后固定不换）；hover 淡出露数字、点击才显示选中圈；`import.meta.glob` 自动扫描 `src/assets/calendar-overlays/`，往目录加图即自动纳入随机池。
- **输入草稿持久化**：对话输入框未发送的文字本地留存，切窗口 / 重挂载都不丢。
- **「清除所有本地数据」**：设置弹窗底部新增「数据管理」分组，一键清除所有任务、对话记录与各项设置，恢复为初始状态（含二次确认弹窗）。后端 `DELETE /tasks/clear-all` 清空三表并重建默认数据。

### 改进

- **截止时刻口径统一（时间检测基础）**：前端「迫近高亮」与「DDL 提醒」合并复用同一 `dueAtMs` 函数，消除多份实现各自解析日期/时刻的漂移风险——迫近判定、优先级自动升级、系统提醒三项能力共享同一个"此时刻距截止还有多久"的计算结果。后端独立维护一份同名函数（前后端分包无法共享文件），注释锁定口径。
- **优先级自动升级（时间驱动）**：后端定时器按距截止远近自动提升开放任务优先级——剩 ≤24h→急、≤48h→高、≤72h→中，**只升不降、幂等**；前端各视图 60s 轮询刷新。与静态 `priority` 字段解耦：用户/ AI 设定的优先级是"重要度"，实时升级是"紧急度"，二者互不污染。
- **DDL 系统提醒**：任务截止前 24 小时、前 6 小时各弹一次 Web Notification；纯日期任务（无 `due_time`）锚定当天 23:59；超过截止 24h 不再补提醒避免刷屏；`localStorage` 去重防重复弹窗。配合迫近高亮形成"被动提醒 + 主动可见"的双通道时间感知。
- **日历配色 / 布局**：其他月日期统一为中灰弱化；年 / 月改下拉选择（选即切，修掉受控数字输入框「切不动」的 bug）；右侧任务列表收敛为内部滚动、日历卡片底贴窗口底。
- **侧栏 / 设置**：导航换自绘图标；设置入口去文字、齿轮放大、模型名左对齐；设置副标题更新为「所有配置仅保存于本机」；悬浮窗说明补充迫近呼吸预警描述。

## [0.3.0] - 2026-06-23

### 新增

- **迫近高亮**：任务在 DDL 前 2 小时内（含已过期未完成）标题自动变红，实时提示时间紧迫——覆盖悬浮窗近期 / 全部、主窗口全部 / 日历 / 标签、对话页 DDL 时间轴。与静态优先级互不影响（重要度 vs 紧急度分离）。
- **优先级随 DDL 自动升级**：后端定时器（启动 + 每 5 分钟）按距截止远近自动提升开放任务优先级——剩 ≤24h→急、≤48h→高、≤72h→中，**只升不降、幂等**；各视图 60s 轮询刷新。
- **相对时间解析**：支持「十分钟后 / 半小时后 / 2 小时后」等相对说法——给 AI 注入当前精确时刻（含星期），换算为绝对截止时刻，跨零点自动进位次日。

### 改进

- **悬浮球告警**：有迫近任务时，悬浮球**白光晕脉动 + 透明度闪烁 + 上下浮动**；无迫近任务时为同款白光晕 + 半幅浮动的静止态，两态统一大小。
- **透明窗黑底修复（了结遗留）**：用常驻 `filter` 动画持续重绘，修掉 macOS Sequoia 下透明悬浮球**四角偶现黑底**的历史遗留（静止 / 告警两态均不黑）。
- **悬浮球展开提速**：缓存屏幕信息，点击展开不再每次做 `currentMonitor` 测量，更跟手。
- **全局复制**：整个软件内框选复制只取文字本身，不再带入布局空白 / 换行。
- **降级文案**：AI 不可用时的提示去掉历史 Claude Code 措辞，改为「请在设置中配置模型」。

## [0.2.1] - 2026-06-23

### 修复

- **打包版偶发「后台未启动」**：根因是前端与后端 sidecar 的启动竞态 + 端口缓存错误，而非后端真的没起来。
  - 打包版后端走动态端口（`TASKDECK_PORT=0`，由系统分配），Rust 需等 node sidecar 冷启动并上报 `TASKDECK_PORT=<n>` 后才知道实际端口。
  - 旧逻辑：webview 加载快于 sidecar 冷启动，抢先 `invoke("server_port")` 拿到 `null`，却用 `_basePromise ??=` 把这个坏基址（`http://127.0.0.1:null`）**永久缓存**。此后 `waitForHealth` 15 秒内反复请求同一个坏地址，必然超时 → 显示「后台未启动」。表现为时好时坏，取决于谁先就绪。
  - 修复（`app/frontend/src/api/client.ts`）：端口未就绪时 `resolveBase` 返回 `null` 且**不缓存**；`waitForHealth` 每轮重新解析端口，直到拿到真实端口才落定基址并通过健康检查。
