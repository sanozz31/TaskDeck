# 更新记录 / Changelog

本项目版本号遵循 `主.次.修订`。日期为发布当天（本地时区）。

## [1.2.2] - 2026-06-26

> 修复 DDL 系统提醒在桌面版从未触发的根因（接入原生通知通道），并彻底根除日历改期后任务残留旧日期的问题。

### 修复

- **桌面版收不到 DDL 系统提醒**：此前提醒只走浏览器的 `new Notification()`，而 macOS 的 Tauri WKWebView 不提供 Web Notification API——`checkReminders` 开头的权限判断直接静默返回，桌面版用户从未收到过截止提醒，且无任何报错。现接入 Tauri 原生通知插件（`tauri-plugin-notification`），提醒改为**双通道**：桌面端走系统原生通知、浏览器端降级到 Web Notification；并补齐 Rust 插件注册与 `notification:default` 能力授权。首次打开应用会弹系统授权框，允许后即可收到「截止前约 24 小时 / 6 小时」的系统通知。
- **日历改期后任务仍残留旧日期（彻底修复）**：1.2.1 的修法仅在「日历视图内、且任务执行日恰好等于当前查看日、且截止日不等于查看日」这一狭窄条件下才迁移执行日，最常见的「截止日与执行日本就同一天」「在全部任务页改期」等场景全部漏掉，旧日期仍残留任务卡。本版从根上解决：日历的圆点、当日卡片、完成图统计全部改为**只认截止日（`due_date`）**，执行日（`scheduled_date`）退出所有显示逻辑，改期后旧日期不可能再残留。

### 重构

- **下线「执行日（scheduled_date）」维度**：该字段曾由 AI 建任务时推断、却无任何用户可编辑入口，又被日历当作一等显示来源，是改期残留类问题的根源。现比照已下线的 `category`，将其从日历显示、任务排序、AI 输出 schema、后端读写与查询中全部移除；数据库列保留以兼容旧库，应用层不再读写（API 响应也不再返回该字段）。

## [1.2.1] - 2026-06-25

> 日历任务卡日期编辑修复版：解决跨月选日不落点、改期后任务仍留在旧日的问题，并补齐月历外侧日期的数据展示。

### 修复

- **任务卡日期选择器跨月选日**：将任务卡编辑态的原生日期输入替换为应用内小日历。点击小日历中当前月份页里的上月 / 下月日期时，会正确选中该日期、切到对应月份并保留高亮，不再出现只跳月或未选中的情况。
- **改期后任务仍显示在旧日期**：手动修改任务截止日期时，同步更新该任务的执行日期，避免日历视图因旧 `scheduled_date` 仍命中原日期而让任务卡继续残留。
- **大日历外月日期任务为空**：日历页的数据查询范围从“本月 1 日到月底”扩展为“当前可见月历网格”，因此点击当前月视图中的外月日期时，右侧也能显示该日期已有任务。

### 改进

- **任务卡小日历视觉打磨**：小日历弹层增加圆角并下移，日期格与日期行距更紧凑，星期行与日期区保持更清晰的间隔。

## [1.2.0] - 2026-06-24

> 对话卡 ChatGPT 式悬停操作栏：用户气泡可复制 / 编辑重生成 / 删除，AI 卡可重试 / 删除 / 切换历史版本。配套打磨编辑态气泡、回收站清空语义与一批文案。

### 新增

- **对话悬停操作栏（图标）**：鼠标悬停**气泡本体**（蓝色用户气泡 / 白色 AI 卡，含其中任务卡；两侧空白不触发）时，气泡下方淡入一排线性图标按钮，替代原右键 / 长按菜单。
  - **用户气泡**：复制（进剪贴板）、编辑、删除（仅删本条对话记录，二次确认）。
  - **AI 解析卡**：重试（用原输入重新生成）、删除（删整条回复并把其中任务归档至「已完成任务」，二次确认）。
- **编辑即重新生成**：点用户气泡的「编辑」，气泡**原地变为可编辑框**（白底蓝边、尺寸与原气泡一致、宽高随内容扩缩），改完发送即按新内容重新解析。
- **历史版本切换**：重新生成（编辑 / 重试）会把上一版的任务**归档至「已完成任务」**，并可选择是否在对话页保留上一版卡片；保留后 **AI 卡与对应用户气泡下方都出现**「‹ n/m ›」版本切换器，可来回查看每次生成的结果。

### 改进

- **「已完成」弹窗底部按钮改为永久删除**：原「清理所有已完成任务」（再归档）改为「清空所有任务」——一键**永久删除**（硬删、不可恢复）弹窗内全部任务，二次确认按钮写「永久删除」。
- **已完成态标题加删除线**：「已完成」弹窗里绿色 ✓ 的已完成任务，标题显示删除线并转灰，与未完成 / 已归档区分。
- **归档 / 删除文案统一精简**：任务卡归档确认统一为「确定将这条任务归档至『已完成任务』？」；重新生成 / 删除回复等提示统一表述为「归档至『已完成任务』」，不再出现「已归档」「可在『已完成』里恢复或永久删除」等冗长措辞。
- **对话底部间距收口**：带操作栏的结尾靠操作栏占位收口、不再叠加额外底距（滚到底不留大白边）；无操作栏的结尾（AI 转圈 / 纯文本出错）保留底距，避免贴输入框过近。
- **ESC 中断改为全局**：分析 / 重新生成进行中，只要焦点在窗口内按 ESC 即中断，不必先点回输入框（原仅输入框聚焦时生效）。
- **去掉确认弹窗的聚焦高亮**：「重新生成」确认弹窗的「保留此版」不再 `autoFocus`，首次弹出不带蓝色 focus 高亮。

### 重构

- 移除对话气泡的右键 / 长按上下文菜单（`ChatPanel` 的 `menu` 状态、长按计时、`ctx-*` 菜单与样式），由悬停操作栏取代。
- 对话数据模型从单 / 多任务字段升级为 `versions[]` + `activeVersion`（兼容迁移历史 `task`/`tasks` localStorage），承载多版本生成与切换。

### 新增

- **日历「回到今天」按钮**：日期网格下方居中一颗蓝色按钮，一键切回当前年月并选中今天——跨年/跨月时年月下拉同步切回。
- **当日小记**：日历日期网格下方整片区域新增随选中日期切换的小记输入框（标题「当日小记 · 月日 周X」），切到哪天显示哪天的记录。文本按日期存于本机 `localStorage`（`taskdeck.calendar.memos`）；内容清空自动删键、不留空串。

### 改进

- **日历网格整体居中**：月份日期网格由靠左改为在面板内水平居中（两侧留白相等），「回到今天」按钮随之相对面板居中；顶部年月略向右对齐。

### 重构（清零前端严格 lint）

> 本轮把前端遗留的全部 `react-hooks` 严格规则报错（6 error + 1 warning，对齐 React Compiler 的 purity 规则）一次性清零，行为不变。

- **完成日图片改为按日期确定性选图**（`CalendarView`）：原实现首次达成时随机抽图并把结果持久化到 `localStorage`（`taskdeck.calendar.dones`），抽图与落库都发生在 `useEffect` 内的 `setState`，触发级联渲染（`set-state-in-effect`）。现改为纯函数按日期字符串哈希取图——同一天恒定同一张、跨刷新不变，**无需任何状态 / ref / effect / 持久化**。副作用：升级前已达成日期显示的覆盖图标可能换成另一张（纯视觉，无数据影响）；`taskdeck.calendar.dones` 不再使用。
- **标签默认选中改为派生**（`TagView`）：原用 effect 在无选中时 `setActive(首个标签)`；改为渲染期派生 `activeTag = active ?? 排序首个`，去掉 effect，且首帧即选中（不再闪一下空列表）。
- **设置表单回填改为渲染期调整**（`SettingsModal`）：原用 effect 把服务端设置灌进表单 state；改为「记录上次 data + data 变化时渲染期回填」，避免级联渲染，用户编辑不被覆盖。
- **提醒器 / 悬浮窗 ref 同步移出渲染期**（`Reminders`、`usePosture`）：原在渲染函数体里直接写 `ref.current`（`refs` 规则报错）；改为在 `useEffect` 内同步（悬浮窗的同步 effect 置于布局 effect 之前以保证读到最新值），并修掉 cleanup 读取可能过期 ref 的 warning。

### 构建 / CI

- **Windows 构建改为仅手动触发**：`build-windows.yml` 去掉 `push: v* tag` 自动触发（此前每次发版推 tag 都在 windows runner 上空跑一遍 NSIS 打包且持续失败），改为仅 `workflow_dispatch`，需要 Windows 版时再手动跑。macOS 发版只产 dmg。

## [1.1.0] - 2026-06-24

> 任务全字段行内编辑 + 全窗口实时同步：在任意视图直接改任务，改动即时落到每一处。

### 新增

- **任务行内编辑（四视图）**：对话 / 日历 / 标签 / 全部任务页的任务卡，悬停标题后的 ✎ 或双击标题/描述进入编辑态，可改 **标题、描述、标签、优先级、截止日期与时刻**。标题留空放弃改动，仅提交真正变化的字段。
- **标签编辑**：编辑态内已选标签为可删胶囊，标签库里未选项点击快速加上，再以「虚线圈 + 号」展开输入新建标签；输入空时退格删除上一个。新建的标签经 `PATCH /tasks/:id` 一并并入标签库（与创建任务一致）。
- **时间轴 / 悬浮窗标题编辑**：对话页顶部 DDL 时间轴、桌面悬浮窗的任务行均支持双击标题行内改名（Enter / 失焦保存，Esc 取消）。
- **对话输入框自适应高度**：随内容自动增高，最多四行，超出后内部滚动。
- **macOS 菜单栏图标**：常驻系统菜单栏一个 template 图标（按菜单栏明暗自动着色）。左键点击打开主窗口；右键弹出功能菜单——「进入对话 / 查看日历 / 查看标签 / 全部任务」点击直达对应视图并唤起窗口，下方「进入万事 / 退出万事」。配合关闭即隐藏的窗口策略，关掉窗口后可随时从菜单栏唤回。

### 改进

- **跨窗口双向即时同步**：此前仅「主窗口 → 悬浮窗」单向通知，悬浮窗改任务后主窗口需等 60s 轮询。现主窗口新增对称的 `onTasksChanged` 监听，悬浮窗任一改动即时刷新主窗口的时间轴 / 对话 / 各列表页（`BroadcastChannel` 不回投发送方，主窗口收发不自循环）。
- **对话卡实时对齐 + 归档定格**：对话卡片改用实时任务数据按 id 覆盖本地快照，任意来源的改动都即时反映；并持续把实时状态落回快照，任务被归档脱离列表后，对话卡**定格在最后一次状态**而非退回创建时的旧值。
- **对话滚动行为更克制**：仅在「有新消息追加」或「AI 回复刚生成完」时滚到底——编辑卡片不再把视图拽到底；切到别的视图再切回对话页，恢复到离开前的滚动位置（本会话首次进入才落到底看最新）。
- **日历单元格禁止选中/复制**：左侧日期网格设 `user-select: none`，避免误选；右侧任务卡片不受影响。
- **应用显示名改为中文「万事」**：`productName` 由 `TaskDeck` 改为 `万事`，安装后启动台 / 应用程序里显示中文名（`.app` 与 dmg 文件名随之变为「万事」）。`identifier`（`com.taskdeck.desktop`）不变，**老用户数据目录不受影响**。
- **应用图标尺寸对齐系统规范**：原图标画稿仅占画布约 74%，在启动台 / 程序坞里显得偏小；重制为约 80%（四周留 ~10% 安全区），视觉尺寸对齐其它应用。

### 修复

- **行内改名重复提交**：悬浮窗与时间轴的标题输入框同时绑定 Enter 与失焦保存，按 Enter 后输入框卸载会再触发一次 blur 导致重复 `PATCH`。以 `editingRef` 守卫，二次进入直接跳过。
- **多余 effect 触发级联渲染**：移除「非编辑态同步草稿」的冗余 `useEffect`（`react-hooks/set-state-in-effect`），改由进入编辑时初始化草稿，省去无谓重渲染。

## [1.0.2] - 2026-06-23

> 代码审查收敛版 + 热修：`invalidateQueries` 键值不匹配导致时间轴不刷新、Widget 跨窗口通信。

### 修复

- **时间轴不刷新**：TanStack Query v5 的 `invalidateQueries({ queryKey: ["tasks"] })` 默认 `exact: true`，只匹配精确键 `["tasks"]`，无法命中 `["tasks", "all"]`、`["tasks", "completed"]` 等带二级键的查询。修复：全部 3 处失效点（`ChatPanel`、`useTasks`、`CompletedModal`）显式加 `exact: false`。
- **Widget 不即时刷新**：主窗口与悬浮窗各持独立 `QueryClient`，主窗口的 `invalidateQueries` 无法让悬浮窗重新拉取数据，最长延迟 20 秒。修复：新增 `BroadcastChannel` 跨窗口通信（`lib/channel.ts`），主窗口任务变更时即时通知 Widget 侧 `invalidateQueries`。

### 安全

- **CORS 收敛为 allowlist**：服务端不再反射任意 `Origin`，仅放行开发期来源（`TASKDECK_ORIGIN`，默认 `localhost:5173`）与 Tauri webview（`tauri://*`、`tauri.localhost`）。不放行来源不回 `Access-Control-Allow-Origin`、预检直接 `403`——挡掉浏览器中任意网页对本机 loopback 任务库的跨源写操作。

### 改进

- **任务可改截止时刻**：`PATCH /tasks/:id` 字段白名单补入 `due_time`，为后续任务编辑 UI 铺路（此前只能改 `due_date`，时刻会被静默丢弃）。

### 文档

- 三份文档对齐现状：`api.md`（端点全集、`/health` provider 为 `deepseek`、删除已下线的 `TASKDECK_PROVIDER/MODEL` 环境变量）、`data-model.md`（补 `due_time`/`completed_at`、`category` 标 deprecated、新增 `tag_defs`/`settings` 表）、`architecture.md`（AI 降级 tag 修正为「待整理」）。

## [0.2.1] - 2026-06-23

### 修复

- **打包版偶发「后台未启动」**：根因是前端与后端 sidecar 的启动竞态 + 端口缓存错误，而非后端真的没起来。
  - 打包版后端走动态端口（`TASKDECK_PORT=0`，由系统分配），Rust 需等 node sidecar 冷启动并上报 `TASKDECK_PORT=<n>` 后才知道实际端口。

  - 修复（`app/frontend/src/api/client.ts`）：端口未就绪时 `resolveBase` 返回 `null` 且**不缓存**；`waitForHealth` 每轮重新解析端口，直到拿到真实端口才落定基址并通过健康检查。
